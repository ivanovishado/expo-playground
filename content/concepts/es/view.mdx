# Componente View

`View` es una **caja invisible** que agrupa y posiciona otros elementos en la pantalla. No muestra nada por si misma — piensa en ella como un estante que sostiene y organiza tu contenido.

```tsx
<View style={styles.container}>
  <Text>Dentro de un View</Text>
</View>
```

## View vs. div

Si conoces HTML, piensa en `View` como el equivalente en React Native de `<div>`. Pero hay diferencias clave:

| HTML `<div>`                 | React Native `<View>`                |
| ---------------------------- | ------------------------------------ |
| Usa CSS para el diseno       | Usa Flexbox por defecto              |
| Diseno de bloque por defecto | Flex columna por defecto             |
| Renderiza texto directamente | No puede contener texto sin envolver |

**Importante:** No puedes colocar texto sin envolver dentro de un `View` — siempre envuelve el texto en un componente `<Text>`.

```tsx
// Esto fallara
<View>Hola</View>

// Esto funciona
<View><Text>Hola</Text></View>
```

## Anidando views

Construye disenos complejos anidando Views:

```tsx
<View style={{ flex: 1 }}>
  <View style={{ flexDirection: "row", justifyContent: "space-between" }}>
    <Text>Izquierda</Text>
    <Text>Derecha</Text>
  </View>
  <View style={{ flex: 1, alignItems: "center" }}>
    <Text>Contenido centrado</Text>
  </View>
</View>
```

> **Intenta esto:** Crea un diseno con una fila de encabezado en la parte superior y contenido centrado debajo usando Views anidados.
